# 2026-07-22 — 같은 디자인 원칙의 구현체를 한 저장소에서 관리하기

## 문제

원칙 문서, 토큰, 아이콘, 폰트, Primitive와 브랜드 컴포넌트는 같은 디자인 시스템에 속하지만 서로 다른 제품처럼 발전하고 배포될 수 있어, 하나의 저장소에 둘지 분리할지 판단이 필요했다.

## 접근 방식

저장소를 분리하면 제품별 버전과 도구를 독립적으로 운영하기 쉽지만, 하나의 디자인 원칙에서 파생되는 변경을 함께 검토하고 검증하기 어려워질 수 있다. 하나의 저장소에 두면 일관성과 변경 추적에는 유리하지만 제품이 늘어날수록 릴리스와 의존 관계 관리가 복잡해진다.

## 해결

현재 Terra의 자산과 구현체는 동일한 제작자가 같은 디자인 원칙 아래 함께 발전시키며, 관련 변경을 한 흐름에서 검증할 필요가 크다. 따라서 서로 다른 배포물과 버전을 가질 수 있더라도, 동일한 Terra 원칙을 따르는 세부 구현체는 하나의 저장소에서 관리하기로 했다.
