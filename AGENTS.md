# Repository Agent Guidelines

## Journey 작성 원칙

`docs/journey/`는 작업 목록이나 완성된 명세가 아니라, 프로젝트를 진행하며 내린 중요한 선택의 맥락을 보존하는 결정 기록이다.

- 결정 하나를 Markdown 파일 하나로 작성한다. 서로 다른 문제와 선택을 한 파일에 합치지 않는다.
- 파일명은 `YYYY-MM-DD-제목.md`, 문서 제목은 `# YYYY-MM-DD — 제목` 형식으로 작성한다. 순번은 붙이지 않는다.
- 각 기록은 다음 세 항목으로 구성한다.
  1. `문제`: 결정이 필요해진 배경, 제약, 영향을 구체적으로 적는다.
  2. `접근 방식`: 검토한 대안, 조사나 실험, 장단점과 상충 관계, 대안을 제외한 이유를 적는다.
  3. `해결`: 최종 선택과 근거, 선택으로 생기는 영향이나 경계를 적는다.
- 대화와 프로젝트 자료가 뒷받침하는 범위에서 판단 과정을 충분히 이해할 수 있도록 자세히 작성한다. 근거나 대안을 추측해서 추가하지 않는다.
- 선택한 결과만 적지 말고, 근거와 선택 사이의 인과관계가 드러나게 작성한다.
- 세부 명세, 구현 체크리스트, 앞으로 할 일은 중복해서 적지 않는다. 해당 내용은 `CHARTER.md`, `TODO.md` 등 기준 문서에 기록한다.
- 기존 결정을 나중에 변경했다면 과거 파일을 현재 관점으로 고치지 않고, 변경 이유와 새로운 선택을 별도 파일로 추가한다.
