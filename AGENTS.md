# Repository Agent Guidelines

## Journey 작성 원칙

`docs/journey.md`는 작업 목록이나 완성된 명세가 아니라, 프로젝트를 진행하며 내린 중요한 선택의 맥락을 보존하는 결정 기록이다.

- 하나의 기록에는 하나의 문제와 선택만 다룬다. 서로 다른 결정은 별도의 기록으로 나눈다.
- 각 기록은 다음 세 항목으로 구성한다.
  1. `마주한 문제`
  2. `어떤 고민을 했는지`
  3. `어떤 근거로 어떤 선택을 했는지`
- 선택한 결과만 적지 말고, 선택이 필요했던 문제와 고려한 대안을 함께 적는다.
- 근거와 선택 사이의 인과관계가 드러나도록 간결하게 작성한다.
- 세부 명세, 구현 체크리스트, 앞으로 할 일은 중복해서 적지 않는다. 해당 내용은 `docs/charter.md`, `TODO.md` 등 기준 문서에 기록한다.
- 기존 결정을 나중에 변경했다면 과거 기록을 현재 관점으로 고치지 않고, 변경 이유와 새로운 선택을 별도 기록으로 추가한다.
